# Template: Naming Brainstorm

## Prompt Template

```
## What Needs a Name
[Product/project/feature description]

## Constraints
- Must be available on: [GitHub / PyPI / npm / domain]
- Audience: [who will see and use this name]
- Tone: [technical / friendly / authoritative / playful]
- Length: [1 word / 2 words / acronym / flexible]

## Current Working Name
[If any — explain what's wrong with it]

## Names to Avoid
[Competitors, confusable names, overused patterns]

## Deliverable
Propose 10 names. For each:
- Name and pronunciation
- Rationale (what it signals)
- Domain/registry availability likelihood
- One risk or concern
```

## Direction Patterns

| After R1... | Direct to... |
|-------------|-------------|
| Agent proposed creative/unusual names | "Check registry availability for your top 3. Any trademark concerns?" |
| Agent proposed safe/generic names | "These feel generic. What's a name that would make someone stop scrolling?" |
| Strong convergence (2+ agents picked same name) | "Multiple agents picked [X]. Argue against it — what's the weakness?" |
| All proposals feel similar | "Forget the constraints for a moment. What name would you give this if it were a band? A restaurant? A sports team?" |

## Synthesis Focus

Naming brainstorms benefit most from:
1. **Convergence signals** — if 2 of 3 agents independently suggest the same name, that's a strong signal
2. **Registry verification** — always verify availability before falling in love with a name
3. **Negative space** — the "argue against it" direction often surfaces dealbreakers early
